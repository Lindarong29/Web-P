시맨틱태그 - 교재 2권 8페이지

▶[탄생 배경]
과거 웹사이트 개발자들이 id / class 사용할 때 별도의 규칙없이 자기 마음대로 만들어서 사용
-> 만든 사람을 제외한 제3자가 코딩 파일을 볼 때 어려움 발생
-> 개발자들이 자주 사용하고 어느정도 직관적인 이름으로 웹사이트의 특정 영역을 이름을 정하자
-> 태그화 시킨 것 (특별한 기능 없음. 


▶시맨틱태그는 반드시 사용해야하는 태그는 아니다.
	개인의 취향에 따라 사용하는 태그 ! 
	쓰라고 만든거니까 쓰는 게 좋긴 해.

▶ 모든 시맨틱태그는 display:block 속성 !

▶[시맨틱태그 7가지 종류]
1. header : 웹사이트의 최상단 영역을 지칭하는 말로, "로고"를 포함하고 있는 영역
	◾  head + er = 머리 + <head> 태그 이미 있으니까 er 붙여서 <header>로 사용
	◾ <div id="header"> </div> ,   <div class="header"> </div> 으로 쓰던 애들을 <header></header> 로 사용

	cf. 로고에 사용하기 위해 만들어진 태그 = <h1>.
	     따라서 header 안에 <h1> 태그가 반드시 있어야 해.


2. nav : 웹사이트 안에서 메인 메뉴를 포함하고 있는 영역
	◾  클릭을 했을 때 내가 원하는 페이지로 웹사이트를 이동시켜주는. 
	    이런 모습이 나치 주소를 찍으면 길 안내를 해주는 navigation 같다고 해서 줄임말 nav


3. main : header, nav, footer 를 제외한 모든 시맨틱태그들을 포함하는 영역.

4. aside : 웹사이트의 한쪽 측면에 위치한 광고영역

5. section : main 태그 안에서 내용을 영역별로 나눠주는 역할

6. article : section 태그가 영역을 나눠주면, 그 영역 안에서 본문 내용들을 담당하는 역할.
	◾ section 태그와 article 태그는 세트. main 안에 section으로 영역 나누고 그 안에서 article 로 본문들 작성

7. footer : 웹사이트의 최하단부를 지칭. 사람의 신체부위 중 가장 밑에 위치한 부위 (발. foot)
	◾ header가 로고를 포함하고 있다면, footer는 법적으로 웹사이트에 반드시 공지해야하는 약관들을 포함.
	◾ 이용약관, 개인정보처리방침 등...
